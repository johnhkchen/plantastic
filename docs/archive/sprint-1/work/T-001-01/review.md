## Deviation: `apps/*` Workspace Member

The acceptance criteria say "workspace members for crates/*, apps/*". The implementation uses `members = ["crates/*"]` only.

**Reason**: Cargo workspace globs require every matched directory to contain a Cargo.toml. The `apps/api/` and `apps/viewer/` directories exist (with .gitkeep) but have no Cargo.toml — adding `"apps/*"` to members causes `cargo check` to fail. This is a Cargo constraint, not a design choice.

**Resolution**: When the first app ticket (T-004-01 for Axum API skeleton) creates `apps/api/Cargo.toml`, it should also add `"apps/*"` to the workspace members. This is a one-line change and belongs in that ticket's scope.

## Open Concerns

1. **Dependency versions**: geo 0.29 and geojson 0.24 are not the absolute latest (geo 0.32, geojson 1.0 exist). The pinned versions were chosen for mutual compatibility. Future tickets may want to evaluate upgrading, but that's a separate decision.

2. **web/ directory**: Already contains a SvelteKit project with node_modules/ (not tracked by git due to .gitignore). The .gitkeep created there is redundant. This is harmless.

3. **Cargo.lock**: Generated by `cargo check` but not committed. For library crates, Cargo.lock is conventionally not committed. However, since this is a workspace with apps (future binaries), it should eventually be committed. This can be addressed when the first app is created.

4. **No CI**: No GitHub Actions or CI configuration. This is intentionally out of scope for this ticket but should be an early follow-up.
