## Context

The branded PDF proposal needs a professional layout. Typst renders templates to PDF bytes in-process — no subprocess, fits the Lambda model. This ticket creates the template and the pt-proposal rendering pipeline.

## Acceptance Criteria

- Create `crates/pt-proposal/` crate
- Typst template (`templates/proposal.typ`) with:
  - Header: tenant logo placeholder + company name + date
  - Intro paragraph (from BAML narrative)
  - Three-tier comparison table: zone rows × tier columns, line items, subtotals, grand totals
  - Zone callouts section (from BAML narrative)
  - Closing paragraph + CTA
  - Footer: page numbers, "Generated by Plantastic"
- `render_proposal(data: ProposalDocument) -> Result<Vec<u8>, ProposalError>` function
- ProposalDocument struct merges: Quote (×3) + ProposalContent + TenantBranding
- All dollar amounts come from pt-quote — template renders them directly, never from LLM output
- Unit test: render with mock data → valid PDF (starts with `%PDF-`)
- Test: verify PDF byte count is reasonable (>10KB, <5MB)
- `just check` passes

## Implementation Notes

- Add `typst` and `typst-pdf` to workspace dependencies
- Template receives data as JSON via `sys.inputs`
- Tenant branding: logo_url (optional), primary_color (hex), company_name, phone, email
- The three-tier table is the layout centerpiece — zone-aligned rows like the web UI
- Dollar amounts are strings (pre-formatted by pt-quote with Decimal precision)
- Consider including material photos if photo_ref is present (fetched from S3 as bytes)
