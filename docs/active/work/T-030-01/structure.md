# T-030-01 Structure — Typst Template & PDF Rendering Pipeline

## Files Created

### `crates/pt-proposal/templates/proposal.typ`
Typst template file. Parses JSON from `sys.inputs.data`. Layouts:
- Header block with company name + date
- Intro paragraph
- Three-tier comparison table (zone rows × tier columns)
- Tier narrative sections
- Zone callouts
- Closing paragraph + CTA
- Footer with page numbers and "Generated by Plantastic"

### `crates/pt-proposal/src/render.rs`
New module containing:
- `TenantBranding` struct — branding fields decoupled from DB row
- `ProposalDocument` struct — merges quotes + narrative + branding
- `TemplateData` (private) — intermediate with pre-formatted dollar strings
- `render_proposal(data: ProposalDocument) -> Result<Vec<u8>, ProposalError>`
- Dollar formatting helper: `format_dollars(d: Decimal) -> String`

### `crates/pt-proposal/tests/render_test.rs`
Integration tests:
- `render_produces_valid_pdf` — mock data → PDF starts with `%PDF-`
- `render_pdf_size_reasonable` — >10KB, <5MB
- Both use `pt_test_utils::timed()` for 10s timeout

## Public Interface (render.rs)

```rust
// Types
pub struct TenantBranding { company_name, logo_url?, primary_color?, phone?, email? }
pub struct ProposalDocument { project_name, project_address, date, branding, narrative, good_quote, better_quote, best_quote }

// Function
pub fn render_proposal(data: ProposalDocument) -> Result<Vec<u8>, ProposalError>
```

## Data Flow

```
ProposalDocument
  → to_template_data()     // Decimal → "$X,XXX.XX" strings
  → serde_json::to_string  // → JSON string
  → typst-as-lib World     // sys.inputs.data = json_string
  → typst::compile         // → Document
  → typst_pdf::pdf         // → Vec<u8> PDF bytes
```

## Dependencies (directed)

```
pt-proposal → pt-quote (for Quote, LineItem types in ProposalDocument)
pt-proposal → typst-as-lib (for Typst compilation)
pt-proposal → serde_json (for JSON serialization to template)
pt-proposal → rust_decimal (for dollar formatting)
```

Note: pt-proposal does NOT depend on pt-project or pt-materials directly.
It receives Quote structs (which contain TierLevel, ZoneId, etc.) but only
needs their serialized form. The serde derives on those types handle it.
