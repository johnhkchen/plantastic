# T-030-01 Plan — Typst Template & PDF Rendering Pipeline

## Step 1: Add workspace dependencies and update Cargo.toml

- Add `typst-as-lib = "0.15"` to `[workspace.dependencies]` in root Cargo.toml
- Update `crates/pt-proposal/Cargo.toml`:
  - Add `typst-as-lib.workspace = true`
  - Move `serde_json` from dev-deps to deps
  - Add `rust_decimal.workspace = true`
  - Add `pt-quote = { path = "../pt-quote" }` to deps (for Quote types)
- Verify: `cargo check -p pt-proposal`

## Step 2: Add Render error variant

- Edit `crates/pt-proposal/src/error.rs`
- Add `Render(String)` variant to `ProposalError`
- Verify: `cargo check -p pt-proposal`

## Step 3: Create render.rs with types and function stub

- Create `crates/pt-proposal/src/render.rs`
- Define `TenantBranding` struct (Serialize, Deserialize, Debug, Clone)
- Define `ProposalDocument` struct with fields: project_name, project_address, date, branding, narrative (ProposalContent), good_quote/better_quote/best_quote (Quote)
- Define private `TemplateData` struct mirroring ProposalDocument but with dollar amounts as formatted strings
- Implement `format_dollars(d: Decimal) -> String` — formats with commas and 2 decimal places, prefixed with `$`
- Implement `ProposalDocument::to_template_data(&self) -> TemplateData`
- Stub `render_proposal()` that serializes to JSON and returns placeholder
- Wire into lib.rs: `mod render;` + re-exports
- Verify: `cargo check -p pt-proposal`

## Step 4: Create Typst template

- Create `crates/pt-proposal/templates/proposal.typ`
- Template structure:
  1. Parse `sys.inputs.data` as JSON
  2. Page setup: A4/letter, margins, footer with page numbers
  3. Header: company name (large), date (right), divider line
  4. Project: name + address
  5. Intro paragraph
  6. Three-tier comparison table:
     - Group line items by zone across all three tiers
     - Columns: Zone/Material | Good | Better | Best
     - Subtotals, tax, grand totals row
  7. Tier narratives: headline, description, bullet differentiators
  8. Zone callouts: label + note
  9. Closing paragraph
  10. CTA line
  11. Footer: "Generated by Plantastic"
- Template must handle missing optional fields (logo_url, tax, etc.)

## Step 5: Implement render_proposal()

- Load template via `include_str!("../templates/proposal.typ")`
- Create TypstWrapperWorld with template content
- Set `sys.inputs.data` to serialized TemplateData JSON
- Compile with `typst::compile(&world)`
- Export with `typst_pdf::pdf(&document, ...)`
- Map errors to `ProposalError::Render`
- Verify: `cargo check -p pt-proposal`

## Step 6: Write tests

- Create `crates/pt-proposal/tests/render_test.rs`
- Helper: `mock_proposal_document()` → builds a ProposalDocument with:
  - 2 zones (Patio 12×15, Garden Bed 10×20)
  - 3 tiers with realistic line items and Decimal totals
  - Mock narrative content (can reuse MockProposalGenerator output pattern)
  - Sample TenantBranding
- Test `render_produces_valid_pdf`:
  - Call `render_proposal(mock_document)`
  - Assert result is Ok
  - Assert PDF starts with `%PDF-`
  - Wrapped in `timed()`
- Test `render_pdf_size_reasonable`:
  - Assert len > 10_000 (10KB)
  - Assert len < 5_000_000 (5MB)
  - Wrapped in `timed()`
