## Design Decisions

1. **SatelliteBaseline BAML type** — flattened from ProjectBaseline to keep prompt
   focused. Drops polygon geometry and per-cell sun data; keeps lot area, tree inventory,
   average sun hours.

2. **ReconciledFeature** — new type rather than reusing ClassifiedFeature. Has `source`
   field ("scan"/"satellite"/"both") that ClassifiedFeature lacks.

3. **RecommendedZone** — extends SuggestedZone concept with sun_exposure_hours and
   data_sources attribution.

4. **convert.rs** — bridge between pt-satellite types and BAML types. Lives in
   pt-reconciler since it's a presentation concern.

## Open Concerns

1. **No scenario advancement.** This ticket builds BAML infrastructure but doesn't
   directly flip a scenario to green. The reconciliation capability will be exercised
   when an end-to-end scan → analyze → reconcile pipeline scenario is created.

2. **Satellite data coverage.** EmbeddedSource only covers a few SF addresses. Powell &
   Market isn't one of them, so the mock fixture is self-consistent but can't be
   validated against real satellite data for that location.

3. **Prompt token budget.** With three full data sources, the prompt is the largest in
   the system. Monitor token usage when running against real LLM to ensure it stays
   within Haiku's context window. The SatelliteBaseline summary helps, but large scan
   feature sets (10+ features) could push limits.
