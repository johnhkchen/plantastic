# T-034-01 Research: BAML ClassifyFeatures

## Current State

### FeatureCandidate (T-033-02, complete)

`crates/pt-scan/src/feature.rs` defines `FeatureCandidate` — the geometric summary
the LLM will classify. Fields:

| Field             | Type     | Description                                          |
|-------------------|----------|------------------------------------------------------|
| `cluster_id`      | `usize`  | DBSCAN cluster ID                                    |
| `centroid`        | `[f64;3]`| XYZ meters                                           |
| `bbox_min/max`    | `[f64;3]`| Bounding box corners                                 |
| `height_ft`       | `f64`    | Max ground-plane distance, feet                      |
| `spread_ft`       | `f64`    | Max XY extent, feet                                  |
| `point_count`     | `usize`  | Points in cluster                                    |
| `dominant_color`  | `String` | "green"/"brown"/"gray"/"white"/"mixed"/"unknown"     |
| `vertical_profile`| `String` | "columnar"/"flat"/"conical"/"spreading"/"irregular"   |
| `density`         | `f64`    | Points per cubic meter                               |

Extraction: `extract_candidates(clusters, points, ground_plane) -> Vec<FeatureCandidate>`
Exported from `pt_scan::extract_candidates`.

### BAML Infrastructure

- **Generator**: `baml_src/generators.baml` — Rust output, async mode, v0.218.0
- **Clients**: `baml_src/clients.baml` — `ProposalWriter` (Sonnet 4), `ProposalWriterFast` (Haiku 4.5), `ProposalFallback` (try fast then full)
- **Existing function**: `baml_src/proposal.baml` — `GenerateProposalNarrative`
- **Generated code**: `baml_client/` — auto-generated by `baml-cli generate`, includes `B` singleton, function structs, types
- **Workspace dep**: `baml = "0.218.0"` in root `Cargo.toml`

### Proposal Crate Pattern (pt-proposal)

`crates/pt-proposal/` is the template for this work:

1. **`lib.rs`**: `#[path = "../../../baml_client/mod.rs"] mod baml_client;` — includes generated code via path attribute, suppresses lint warnings
2. **`generator.rs`**: Trait `ProposalNarrativeGenerator` (async_trait), `BamlProposalGenerator` calling `crate::B.GenerateProposalNarrative.call()`
3. **`mock.rs`**: `MockProposalGenerator` — deterministic fixture-based output, `MockFailingGenerator` for error paths
4. **`claude_cli.rs`**: `ClaudeCliGenerator` — builds prompt manually, calls `claude` CLI with env-stripped subprocess, parses response with BAML
5. **`error.rs`**: `ProposalError` enum via thiserror
6. **Re-exports**: `pub use baml_client::types::*` for generated types, `pub use baml_client::B`

### Pipeline Integration

`crates/pt-scan/examples/process_sample.rs` runs 6 stages: parse → downsample → outlier filter → RANSAC → terrain export → feature extraction. Classification would be stage 7, but lives in a separate crate since it requires BAML/async.

### Test Infrastructure

- **Synthetic**: `tests/integration.rs` builds PLY in memory with `make_synthetic_ply()`
- **Real scan**: `assets/scans/samples/Scan at 09.23.ply` (294MB, Powell & Market)
- **Powell & Market tests**: `test_powell_market_two_clusters()` and `test_powell_market_candidates()` validate clustering and feature extraction on real data
- **Test timeout**: `pt_test_utils::timed()` wrapper, 10s default

### Constraints

- BAML types must mirror but not depend on Rust types — BAML generates its own structs
- `baml_client/` is generated code, checked in, shared via `#[path]` across crates
- No real LLM calls in tests (project rule: mock external services in CI)
- Single BAML request, <5KB input JSON
- `just check` must pass

## Observations

1. The `#[path]` include pattern for `baml_client` means any crate that uses BAML must include the same boilerplate module declaration with lint suppressions.
2. BAML generates its own struct types (with `BamlEncode`/`BamlDecode`) — these are distinct from the domain Rust types. Conversion between `pt_scan::FeatureCandidate` and `baml_client::types::FeatureCandidate` will be needed.
3. The classifier crate needs async (BAML client is async), so it can't live inside pt-scan (which is sync).
4. The `ProposalFallback` client is already configured with retry + fallback — reusable for classification.
5. Powell & Market produces multiple clusters (trees, poles, curbs). The test must validate the two largest are classified as trees — not that all clusters are trees.
