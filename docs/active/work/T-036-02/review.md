## Key Design Decisions

1. **New crate (pt-planter)** rather than extending pt-features — different concern (classify vs. recommend)
2. **Area split evenly** among plant selections per style — simplest fair allocation
3. **Keyword-based price inference** rather than LLM-computed pricing — deterministic, cheap
4. **S.2.3 reuse** rather than new S.4.1 — scenario already existed as NotImplemented placeholder

## Open Concerns

1. **pt-analyzer blocks `just check`**: T-035-01 (concurrent ticket) left an incomplete pt-analyzer crate with compile errors in `analyzer.rs` (BAML Image API changed). This blocks workspace-wide `just check` but is not introduced by this ticket. Filed for T-035-01 to fix.

2. **Area split strategy**: Currently area is split evenly among plant selections. Real landscape design would use primary/filler/accent proportions (60/30/10). This is a follow-up enhancement.

3. **Price inference is basic**: Keyword matching on common_name works for the mock fixture but may miss unusual plant names. A category field on PlantSelection (grass/annual/succulent/perennial/tree) would be more robust. Consider adding to planter.baml.

4. **LLM not validated against real call**: BamlPlanterEstimator compiles but hasn't been tested with a real ANTHROPIC_API_KEY call. The prompt may need tuning for response quality. The ClaudeCliEstimator is the development path for validation.
