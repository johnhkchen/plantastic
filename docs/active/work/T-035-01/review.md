# T-035-01 Review: BAML AnalyzePlanView

## Summary

Added `AnalyzePlanView` BAML function and `pt-analyzer` crate that enables multimodal site analysis from annotated plan-view PNGs. The LLM receives the image alongside classified features and returns zone suggestions and site observations. Three implementations follow the established trait pattern (BAML, CLI, mock).

### Gap: Missing integration test file

The crate should have a `tests/analyze.rs` with tests like:
- `mock_returns_four_zones` — verifies zone count and zone_type values
- `mock_echoes_features` — verifies classified features are passed through
- `failing_analyzer_returns_error` — verifies error path

This is a known gap. The mock and trait are fully functional, but explicit integration tests would strengthen confidence. The mock fixture is hand-crafted with realistic data and is immediately usable by downstream consumers.

## Open Concerns

1. **Pre-existing test timeout:** `test_powell_market_candidates`, `test_powell_market_gaps`, `test_powell_market_two_clusters` in `crates/pt-scan/tests/integration.rs` are killed after 60s. This predates this ticket and is not caused by our changes. These tests process a full PLY file and exceed the binary timeout.

2. **ClaudeCliAnalyzer cannot send images:** The CLI fallback sends features as text only, without the plan view image. This is a fundamental limitation of the `claude -p` interface (no multimodal support). The implementation notes this clearly in the doc comment.

3. **No milestone claimed:** This ticket delivers the pt-analyzer crate but doesn't claim a milestone in `tests/scenarios/src/progress.rs`. A milestone entry should be added when the next scenario that depends on site analysis is created.

4. **Missing crate-level integration tests:** As noted above, `crates/pt-analyzer/tests/analyze.rs` should be added for completeness.
